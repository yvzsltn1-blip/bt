# Session Changes — 2026-05-03

## 1. Optimizer Kayıt Permission-Denied Hatası

**Sorun:** `bt-analiz.web.app/optimizer.html` sayfasında "Onayla ve Kaydet" butonuna basıldığında Firestore `permission-denied` hatası alınıyordu. Yerel kural doğrulaması geçiyordu ama Firestore isteği reddediliyordu.

**Kök neden:** Firestore kurallarındaki `isAdmin()` kontrolü `request.auth.token.email == "yavuz@gmail.com"` (büyük/küçük harf duyarlı) kullanıyordu. Yeni kurallar bu kontrolü `matches('(?i)^yavuz@gmail\\.com$')` (büyük/küçük harf duyarsız) olarak güncelledi ve `approvedStrategies` koleksiyonu için write kuralından `isAdmin() &&` koşulunu kaldırdı — veri doğrulaması yeterli.

**Çözüm:** `firestore.rules` dosyası zaten güncellenmiş durumdaydı; `firebase deploy --only firestore:rules` ile deploy edildi.

## 2. Optimizer Kayıt Formatı → Simülasyon Formatı

**Sorun:** Optimizer'dan kaydedilen onaylı strateji girişleri, simülasyon kayıtlarından farklı bir format kullanıyordu. Kullanıcı her iki kaynaktan gelen kayıtların aynı formatta olmasını ve ek olarak kademe (stage) bilgisi içermesini istedi.

### Değişiklikler

#### `optimizer.js`

**`createSavedEntry` fonksiyonu** — optimizer formatından simülasyon formatına geçirildi:

| Alan | Eski (optimizer format) | Yeni (simulation format) |
|---|---|---|
| `source` | `"optimizer"` | `"simulation"` |
| `sourceLabel` | `"Optimizer"` | `"Optimizer"` (korundu) |
| `stage` | vardı | ✅ korundu |
| `allyCounts` | `recommendationCounts` adıyla | ✅ `allyCounts` olarak eklendi |
| `matchSignature` | yoktu | ✅ `buildOptimizerMatchSignature(stage, enemy, ally)` |
| `variantSignature` | yoktu | `"optimizer"` (sabit) |
| `variantTitle` | yoktu | `"Optimizer Onerisi"` |
| `winner` | yoktu | `"ally"` (optimizer sadece kazananları kaydeder) |
| `probabilityBasisPoints` | yoktu | `Math.round(winRate * 10000)` |
| `summaryText` | yoktu | ✅ örnek savaştan (`battleView.summaryText`) |
| `logText` | yoktu | ✅ örnek savaştan (`battleView.logText`) |
| `usedCapacity` | yoktu | ✅ `sampleBattle.usedCapacity` |
| ~~`mode`~~ | vardı | kaldırıldı |
| ~~`objective`~~ | vardı | kaldırıldı |
| ~~`diversityMode`~~ | vardı | kaldırıldı |
| ~~`stoneMode`~~ | vardı | kaldırıldı |
| ~~`modeLabel`~~ | vardı | kaldırıldı |
| ~~`enemySignature`~~ | vardı | kaldırıldı |
| ~~`allyPool`~~ | vardı | kaldırıldı |
| ~~`recommendationCounts`~~ | vardı | kaldırıldı |
| ~~`winRate`~~ | vardı | kaldırıldı (`probabilityBasisPoints` ile değiştirildi) |

**`currentApprovedCandidate` nesnesi** — `battleView` alanı eklendi (renderBattleLog'dan gelen summaryText/logText'i `createSavedEntry` içinde yeniden hesaplamaktan kaçınmak için):
```js
currentApprovedCandidate = { ..., result, battleView };
```

**`renderMatchedSavedStrategy` fonksiyonu** — hem eski optimizer formatını hem yeni simülasyon formatını destekleyecek şekilde güncellendi:
- Eşleştirme: simülasyon formatı için `stage + enemyCounts` üzerinden hesaplanan imza kullanılıyor (eski `enemySignature` alanı yerine)
- `recCounts`: `allyCounts` (sim format) veya `recommendationCounts` (optimizer format)
- `displayWinRate`: `probabilityBasisPoints / 100` (sim format) veya `winRate` (optimizer format)
- `displayModeLabel`: `modeLabel` (optimizer) veya `sourceLabel` (sim)

**`restoreFromQuery` fonksiyonu** — simülasyon formatı desteği eklendi:
```js
// Önce: item.recommendationCounts[unit.key] → hata veriyordu
// Sonra:
item.allyPool?.[unit.key] || item.allyCounts?.[unit.key] || item.recommendationCounts?.[unit.key] || 0
```

#### `saved.js`

Simülasyon formatındaki kayıtların başlık satırında "Simulasyon" sabit etiketi yerine `sourceLabel` gösterilecek şekilde güncellendi. Optimizer kayıtları artık "Optimizer", simülasyon kayıtları "Simulasyon" olarak görünüyor:

```js
// Önce:
`${item.stage ? `${item.stage}. Kademe / ` : ""}Simulasyon / ${formatDate(item.savedAt)}`

// Sonra:
`${item.stage ? `${item.stage}. Kademe / ` : ""}${item.sourceLabel || "Simulasyon"} / ${formatDate(item.savedAt)}`
```
